Power Platform & Dataverse

Storage & Administration

An illustrated guide

What is the Power Platform?

  • Power Apps: Build custom apps for your business needs.
  • Power Automate: Create automated workflows and processes.
  • Power BI: Deliver insights and enable informed decisions.
  • Power Virtual Agents: Create powerful chatbots without code.

At the heart of the platform lies Dataverse.

What is Dataverse?

More than a database, it's a scalable and intelligent data service.

  • Tables: Model and manage business data (e.g., Accounts, Contacts).
  • Columns & Rows: Define the structure and hold the data.
  • Rich Features: Supports complex relationships, business logic, and robust security.

Dataverse Storage: The Three Types

Database

Structured, transactional data. (e.g., Account records, business process flows).

Stored in Azure SQL.

File

Unstructured data and attachments. (e.g., PDFs, images in notes, email attachments).

Stored in Azure Blob Storage.

Log

Change logs and audit trails. (e.g., Record updates, deletions, system job logs).

Optimized for audit data.

How to Check Your Storage

  1. Navigate to the Power Platform Admin Center (PPAC).
  2. Go to Resources > Capacity.
  3. Review the Summary tab for a tenant-wide overview.
  4. Select an environment to drill down into details.
[Illustration: Screenshot of the PPAC Capacity summary page]

Drilling Down into Storage Details

In the environment view, check the tabs:

  • Database Tab: Shows top tables by data size.
  • File Tab: Shows top tables using file storage (often Notes or Attachments).
  • Log Tab: Shows top tables consuming log capacity.
[Illustration: Screenshot of environment-level storage drill-down]

Common Causes of High Storage Usage

  • Database: Old system jobs (`AsyncOperationBase`), wide tables, verbose auditing.
  • File: Email attachments (`Attachment`) and notes with attachments (`Annotation`).
  • Log: Over-auditing of high-transaction tables.

Administration & Cleanup Strategies

  • Use Bulk Delete Jobs: Create recurring jobs to clean up completed system jobs and old audit logs. This is your most important tool.
  • Manage Attachments: Educate users and consider using SharePoint integration for document storage.
  • Optimize Auditing: Be selective about what you audit and review retention periods.
[Illustration: Screenshot of setting up a bulk delete job]

PowerShell for Administration

Automate and report on storage using PowerShell cmdlets.

# 1. Install the module (one-time setup)
Install-Module -Name Microsoft.PowerApps.Administration.PowerShell

# 2. Connect to your tenant
Add-PowerAppsAccount

# 3. Get tenant-wide capacity
Get-AdminPowerAppCapacity

# 4. Get consumption for an environment
Get-AdminPowerAppEnvironmentConsumption -EnvironmentName <YourEnvName>

A powerful tool for scripting and automation.

Capacity Management & Add-ons

  • You will get notifications when you approach storage limits.
  • There is a grace period, but admin operations may become restricted.
  • If optimization isn't enough, you can purchase capacity add-ons via the Microsoft 365 admin center.

Summary & Key Takeaways

  1. Be Proactive: Regularly monitor your storage.
  2. Know the Types: Understand Database, File, and Log capacity.
  3. Use the Tools: Leverage Bulk Delete Jobs and PowerShell.
  4. Govern Your Data: Establish a clear governance plan.
  5. Optimize First: Try to clean up before buying more capacity.
